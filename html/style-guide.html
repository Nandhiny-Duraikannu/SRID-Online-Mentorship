Style Guide

About

The styleguide is a resource for designers, product managers, and developers, providing a common language around UI patterns.

To develop this system we have used Materialize.

Materialize is a design language that combines the classic principles of successful design along with innovation and technology.

Our goal is to develop a system of design that allows for a unified user experience across all their products on any platform.

We use it to maintain modular front-end code and visual consistency across the web app.

Buttons

Basics

Buttons are used as triggers for actions.
Depending on the use case, buttons contain a label and/or an icon.
There are a variety of styles, sizes, and variations that can be used for different situations.
All button labels are sentence case.
They should be as short as possible while clearly explaining what will happen when the button is clicked.

Size

Large, use the btn-large class.
Normal, use the btn col s12

Forms

Basics

Properly mark fields as required
Add placeholders when necessary, especially when users may not fully understand the purpose of the input
If necessary, place short, related inputs on the same line
Ensure that unsaved changes are not easily lost

Field Width

Field widths should communicate the expected or max length of content.
Use consistent, non-arbitrary widths to help with alignment.
These widths apply to similar controls like selects, date inputs, and sliders.
These widths may not be appropriate in some situations or layouts.

We recommend using 'input-field col s12'

Icons

Basics

System icons should always be one solid color
System icons should either be black, white, or any other primary UI color
System icons can be placed in colored circles

Sizes

Icons can vary in size
When scaling icons, ensure the sizes used appear sharp on standard and high DPI screens
